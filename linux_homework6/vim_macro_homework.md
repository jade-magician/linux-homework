# Vim 宏生成 SSH 命令列表的方法

## 作业题目

使用 Vim 的宏功能，生成从 `ssh root@10.64.198.1 > 1.log` 到 `ssh root@10.64.198.255 > 6.log` 的命令列表，其中日志文件编号按以下规则分配：
- 1~50: > 1.log
- 51~100: > 2.log
- 101~150: > 3.log
- 151~200: > 4.log
- 201~250: > 5.log
- 251~255: > 6.log

## 方法概述

本方法使用 Vim 的宏功能来自动化生成重复的 SSH 命令。首先手动输入第一行，然后录制宏来复制行并递增 IP 地址，最后使用替换命令修正日志文件编号。

## 详细步骤

### 1. 打开 Vim 并输入第一行

在终端运行：

```bash
vim ssh_commands.txt
```

按 `i` 进入插入模式，输入：

```
ssh root@10.64.198.1 > 1.log
```

按 `<Esc>` 退出插入模式。

### 2. 录制宏

按 `qq` 开始录制宏到寄存器 `q`。

录制内容：
- `yyp`：复制当前行并粘贴到下一行
- `f.`：定位到 IP 中的第一个点
- `f.`：定位到第二个点
- `f.`：定位到第三个点
- `l`：向右移动到最后一个 IP 段的数字
- `<C-a>`：递增数字（按住 Ctrl 键，按 a 键）

按 `q` 结束录制。

宏的完整按键序列：`qq yyp f. f. f. l <C-a> q`

### 3. 执行宏生成 255 行

按 `254@q` 执行宏 254 次，生成从 `10.64.198.1` 到 `10.64.198.255` 的 255 行，每行日志都是 `> 1.log`。

### 4. 修正日志文件编号

使用 Vim 的替换命令修正日志编号：

```vim
:51,100s/> 1\.log/> 2.log/
:101,150s/> 1\.log/> 3.log/
:151,200s/> 1\.log/> 4.log/
:201,250s/> 1\.log/> 5.log/
:251,255s/> 1\.log/> 6.log/
```

这些命令将指定行范围内的 `> 1.log` 替换为对应的日志编号。

### 5. 保存文件

按 `:w` 保存文件。

### 6. 验证结果

按 `:echo line('$')` 查看总行数，应为 255。

查看开头几行：`:1,5p`

查看中间几行：`:48,52p`

查看结尾几行：`:251,255p`

## 结果示例

生成的文件内容如下：

```
ssh root@10.64.198.1 > 1.log
ssh root@10.64.198.2 > 1.log
...
ssh root@10.64.198.50 > 1.log
ssh root@10.64.198.51 > 2.log
ssh root@10.64.198.52 > 2.log
...
ssh root@10.64.198.100 > 2.log
ssh root@10.64.198.101 > 3.log
...
ssh root@10.64.198.250 > 5.log
ssh root@10.64.198.251 > 6.log
ssh root@10.64.198.252 > 6.log
ssh root@10.64.198.253 > 6.log
ssh root@10.64.198.254 > 6.log
ssh root@10.64.198.255 > 6.log
```

## 关键技术点

- **宏录制**：Vim 宏可以录制一系列操作，用于重复执行。
- **<C-a>**：在 normal 模式下递增光标下的数字。
- **替换命令**：`:s` 用于字符串替换，支持行范围。
- **正则表达式**：在替换中使用 `\` 转义点号。

## 优点

- 自动化生成大量重复内容，提高效率。
- 结合宏和替换，灵活处理复杂规则。
- 纯 Vim 操作，无需外部脚本。

## 注意事项

- 确保在 normal 模式下按 `<C-a>`，否则可能进入插入模式。
- 替换命令中的行范围需要精确计算。
- 如果 `<C-a>` 不工作，可以用 `:s/\v(\d+)\ze\s*>\s*\d+\.log$/\=submatch(1)+1/` 替换。

此方法展示了 Vim 宏和替换的强大功能，适合批量文本处理任务。
